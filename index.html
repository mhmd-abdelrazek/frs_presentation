02 — Architecture

High-level Overview

Three core components communicate to capture, store, and display attendance data.

🖥️
ESP32
Microcontroller + Fingerprint sensor. Reads biometric data on-site.
Wi-Fi / HTTP
☁️
Firestore
Cloud database by Firebase. Stores users, sessions & records remotely.
REST / SDK
📱
Mobile App
Flutter app for managing sessions and viewing attendance.

04 — Hardware

ESP32 Stack

Embedded components at the physical attendance terminal.

🖥️
ESP32 Microcontroller
Dual-core 240 MHz processor with built-in Wi-Fi and Bluetooth. Handles sensor communication, local logic, and cloud sync over HTTP/HTTPS to Firebase.
Xtensa LX6 · 240 MHz
Wi-Fi 802.11 b/g/n
Bluetooth 4.2
UART · GPIO
👆
Fingerprint Reader Module
Optical fingerprint sensor (e.g. R307/AS608). Captures, enrolls, and verifies fingerprints on-device. Returns a numeric ID that maps to the Firestore users collection.
UART Interface
Up to 127 templates
FAR < 0.001%
~1 s recognition
